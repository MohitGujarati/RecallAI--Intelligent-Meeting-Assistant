# RecallAI - Audio Recording & Transcription App

**Android Developer** | *Kotlin, Jetpack Compose, Coroutines, Flow, Room, MVVM, Hilt (Dagger)*

*   **Jetpack Compose UI:** Architected a fully declarative, fluid UI from scratch using **Jetpack Compose**. Developed complex responsive screens (e.g., `RecordingScreen`, `DashboardScreen`, `MeetingDetailScreen`, and `ChatTabContent`) using `@Composable` functions, eliminating XML layouts and ensuring consistent dynamic theming (`RecallTheme`).
*   **Reactive State with ViewModel Overlay:** Implemented a robust MVVM architecture utilizing **ViewModel** overlay (`DashboardViewModel`, `MeetingDetailViewModel`). Effectively bridged the UI and data layers by managing business logic independently and exposing immutable `StateFlow` to the Compose UI for seamless, reactive view updates without tight coupling.
*   **Asynchronous Data Streams (Flow/StateFlow/SharedFlow):** Engineered an extensive real-time reactive architecture using Kotlin **Flow**. 
    *   Exposed continuous UI states using `StateFlow` (e.g., `RecordingUiState`).
    *   Emitted one-off critical system events (Audio Focus, Silence Detection, Phone Call interruptions) via `SharedFlow`.
    *   Wrapped streaming network APIs (Gemini AI responses) with `callbackFlow` and `channelFlow` for concurrent transcription and AI chat.
*   **Structured Concurrency with Coroutines:** Leveraged Kotlin **Coroutines** (`viewModelScope`, `Dispatchers.IO`) for efficient, non-blocking background processing. Managed complex concurrency such as chunking live audio data, delegating network API calls (Gemini Streaming), and executing asynchronous database CRUD operations.
*   **Local Persistence using Room:** Engineered a highly responsive offline-first local database using **Room** (`AppDatabase`). Designed normalized schemas and DAOs for critical app entities (`Meeting`, `Transcript`, `Summary`, `ChatMessage`, `AudioChunk`). Utilized Room's native SQL-to-Flow reactive queries (`observeAllMeetings`, `observeTranscriptsForMeeting`) to instantly reflect database updates in the Compose UI.
*   **Dependency Injection with Hilt:** Architected a modular, easily testable codebase utilizing **Hilt (Dagger)**. Centralized dependency management through specialized modules (`ServiceModule`, `RepositoryModule`, `DatabaseModule`, `NetworkModule`). Scoped and injected critical dependencies like `AppDatabase`, repositories, Retrofit networking services, and `WorkManager` configurations directly into ViewModels and Android components, significantly reducing tightly-coupled boilerplate code.
